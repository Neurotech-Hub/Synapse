Poll now

Fetches approved sources that are not hidden. Includes RSS feeds and HTML monitors. Refreshing ingest gives persona builders something to read on the identities side — if items are empty, run this before rebuilding. After you start a run, progress appears in the Polling & ingest card with a shortened host/path for the source in progress (or the next queued one), plus the bar; Poll now stays disabled meanwhile.

Stale personas vs polling

Editing sources, subjects, or ownership marks persona rows stale without auto-running heavy LLM work. Persona rebuilds read ingested content from owned sources — if everything is still empty, use Poll now first. The identities card labels rows Polling first until at least one content item exists on owned eligible sources. Rebuild all runs a bounded batch of poll-ready stale subjects each click — batch size follows SYNAPSE_DASH_IDENTITY_BATCH (or the source-save identity burst fallback) — skips Polling first rows — and is disabled when the stale count is zero or while a poll is running. Use per-row Rebuild for individual subjects anytime.

Lead reports

Sources awaiting approval

URLs submitted from the public site appear here until you approve them — open each source with View to assign… to set an owner if needed and use Approve for polling there.
