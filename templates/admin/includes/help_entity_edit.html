Display name & slug

Display name is what you edit: titles, departments, punctuation (“Washington University Neurotechnology Lab”). The form shows a live slug preview derived from this text.

Slug is generated from the display name. On save we pick a URL-safe slug—lowercase Latin letters and digits plus _/-—and use it in prompt catalogs (primary_entity_slug) and dedup. If that slug collides with an existing row, we append _2, _3, and so on.

Editing the display name regenerates the slug on save — if you rename something that other leads reference, fingerprints and LLM resolution may need a follow‑up refresh.

Kind & linked data

Choose the type that best matches how you reference the subject in catalogs (lab, person, place, org).

For person rows only, the Persona dashboard (top of Edit entity) snapshots linked sources via Ollama when you choose Rebuild persona — catalog Save does not run it. Institution and Department show after a successful build and seed the next rebuild. Poll/source saves can still enqueue background persona refreshes for linked entities.

PubMed-style RSS usually puts full abstracts in content:encoded; we persist that text on poll. Fetching NIH PubMed URLs uses browser-like headers and prefers curl when installed (otherwise urllib). Disable curl with SYNAPSE_RSS_PUBMED_CURL=0. Persona prompts default to SYNAPSE_OLLAMA_IDENTITY_NUM_CTX=32768 so large feeds are not silently truncated — lower it if your Ollama/model errors on context size, set 0 for server defaults, or pass JSON via SYNAPSE_OLLAMA_IDENTITY_OPTIONS. Evidence packing uses SYNAPSE_IDENTITY_FULL_TEXT_ITEMS, SYNAPSE_IDENTITY_CONTENT_BUDGET_CHARS, SYNAPSE_IDENTITY_CHUNK_MAX_CHARS, SYNAPSE_IDENTITY_PAPER_DAYS.
